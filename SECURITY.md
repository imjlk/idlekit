# Security Policy

## Supported versions

`idlekit` is currently maintained as a v1 project. Security fixes are applied to the latest v1-compatible branch and current npm packages.

## Reporting a vulnerability

Please do not open a public issue for a suspected security problem.

Preferred disclosure path:

1. Use GitHub Security Advisories / private vulnerability reporting if enabled for the repository.
2. If that is not available yet, contact the maintainer through the repository owner account and include:
   - affected package (`@idlekit/money`, `@idlekit/core`, or `@idlekit/cli`)
   - impacted version
   - reproduction steps or proof of concept
   - severity assessment

We will aim to acknowledge a report within 5 business days.
